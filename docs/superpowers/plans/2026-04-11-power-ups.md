# Power-Up System Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add 4 collectible power-ups that spawn in space and modify the ship's weapons or add a shield.

**Architecture:** New `PowerUp` entity with `PowerUpType` enum. `PowerUpSystem` handles spawning, despawning, collection, and active effect tracking. `BulletPool.acquire()` checks active power-up to modify fire pattern. `CollisionSystem` checks ship-vs-powerup and ship-vs-asteroid/saucer (shield). HUD shows active power-up indicator with countdown.

**Tech Stack:** Kotlin, libGDX ShapeRenderer (vector icons), existing event bus pattern.
